## summary
Extended `kbu notebook-init` to route new work-notebook repos to
`~/Dropbox/Projects/NotebookWorkspaces/<group>/<name>` while preserving
backward compatibility for existing repos. Added the `_NOTEBOOK_WORKSPACES`
constant, rewrote `_resolve_repo` with a three-tier bare-name lookup (legacy
path exists → use it; NotebookWorkspaces glob has unique match → use it; else
new-repo path requiring `--group`), added `_validate_group` to reject
empty/whitespace/separator-containing group values, threaded `group` through
`notebook_init()`, and added `--group` to the click command. Updated the test
suite from 50 to 68 tests, adding 18 new cases that cover every resolution
path, error case, and invariant specified in the task.

## caveats
- The `patched_projects` fixture monkeypatches `_DROPBOX_PROJECTS` and
  `_NOTEBOOK_WORKSPACES` at module level using `monkeypatch.setattr`. The
  tests that use `_resolve_repo` or `notebook_init` directly with bare names
  rely on this fixture; tests using full paths (str(tmp_path/...)) do not
  need it.
- The `TestResolveRepo::test_bare_name_resolves_to_dropbox` test was replaced
  with three more accurate tests reflecting the new three-tier resolution
  (legacy-exists, new-repo-with-group, new-repo-no-group-raises). The old test
  was incorrect for the new behavior: bare names for non-existent repos no
  longer silently expand to `~/Dropbox/Projects/<name>`.
- The `--group` option is ignored when `repo` contains a path separator, as
  specified. This is documented in both the module docstring and the click
  command docstring.
- The gaa-model venv was temporarily pointed at this worktree to run tests,
  then restored to the Dropbox KBUtilLib. This is safe since editable installs
  update the `.pth` pointer but leave venv packages untouched.
