## summary

Implemented the `kbu researchos` command group in KBUtilLib, mirroring the existing `kbu beril worktree` feature. The feature scaffolds Research-OS (PyPI `research-os`) studies under a parent project at `<root>/<parent>/<name>/`, rewrites MCP config files to use an absolute path to a shared tooling venv binary (solving the PATH problem), writes a `.code-workspace` file, initializes a git repository, and registers the parent group + study project in the AIAssistant project registry out-of-process. Registry, git, and cursor-open steps are all best-effort (warn-and-continue).

Five new source modules were created and registered in the CLI. 81 hermetic unit tests were written covering all required behaviors. The full existing test suite shows no regressions from our changes (all pre-existing failures confirmed against main).
